# Исправление ошибки 404 на GitHub Pages

## Проблема
Сайт выдает ошибку 404 при загрузке ресурсов на GitHub Pages.

## Решение

### 1. Убедитесь, что GitHub Pages настроен правильно

1. Перейдите в **Settings** → **Pages** вашего репозитория
2. Убедитесь, что:
   - **Source**: выбран "GitHub Actions" (не "Deploy from a branch")
   - **Custom domain**: оставьте пустым (если не используете кастомный домен)

### 2. Проверьте, что workflow выполнился успешно

1. Перейдите в **Actions** в вашем репозитории
2. Убедитесь, что последний workflow выполнился успешно
3. Если есть ошибки, проверьте логи

### 3. Очистите кэш браузера

После деплоя очистите кэш браузера:
- **Chrome/Edge**: Ctrl+Shift+Delete (Windows) или Cmd+Shift+Delete (Mac)
- **Firefox**: Ctrl+Shift+Delete
- Или откройте сайт в режиме инкогнито

### 4. Проверьте URL

Убедитесь, что вы открываете правильный URL:
- **С кастомным доменом**: `https://ваш-домен.ru`
- **Без кастомного домена**: `https://famellow.github.io/Cleandvor/` (обратите внимание на `/` в конце!)

### 5. Если проблема сохраняется

1. Проверьте консоль браузера (F12) на наличие ошибок
2. Проверьте Network tab, чтобы увидеть, какие ресурсы не загружаются
3. Убедитесь, что файл `public/CNAME` содержит только комментарии (если не используете кастомный домен)

### 6. Пересоберите проект

Если проблема не решается, попробуйте:

```bash
# Локально
npm run build
npm run preview

# Или просто запушьте изменения в main, чтобы запустить GitHub Actions заново
git add .
git commit -m "Fix GitHub Pages deployment"
git push origin main
```

## Проверка после исправления

После деплоя проверьте:
1. Главная страница загружается: `https://famellow.github.io/Cleandvor/`
2. Все ресурсы загружаются (проверьте Network tab)
3. Нет ошибок в консоли браузера
4. Форма работает корректно

## Дополнительная информация

- Base path для GitHub Pages: `/Cleandvor/`
- Base path для кастомного домена: `/`
- Файл 404.html автоматически обрабатывает редиректы для SPA
